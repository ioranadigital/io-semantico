## 2. LÓGICA DE NEGOCIO

### Autenticación y roles
- `useAuth` hook: al hacer login con `supabase.auth.signInWithPassword()`, consultar `portal_users` para determinar el rol:
  - Si existe fila con ese `user_id` → `role = 'cliente'` → redirect a `/portal`
  - Si no existe → `role = 'interno'` → redirect a `/dashboard`
- `ProtectedRoute`: si no hay sesión → `/login`. Si rol='cliente' intenta ruta interna → `/portal`. Si rol='interno' intenta `/portal` → `/dashboard`.

### Lead Scoring (cálculo en frontend)
```typescript
function calcLeadScore(lead: Lead): number {
  let s = 0;
  if (lead.budget_range?.includes('>10.000')) s += 40;
  else if (lead.budget_range?.includes('5.000')) s += 30;
  else if (lead.budget_range?.includes('2.000')) s += 20;
  else s += 10;
  if (lead.urgency === 'Alta') s += 30;
  else if (lead.urgency === 'Media') s += 15;
  else s += 5;
  const fitSectors = ['Fintech','Logística','SaaS','eCommerce','Retail'];
  if (fitSectors.includes(lead.sector || '')) s += 20;
  else s += 10;
  if (lead.audit_status === 'done' && lead.audit_speed)
    s += Math.round(lead.audit_speed / 10);
  return Math.min(s, 100);
}
```

### Conversión Lead → Cliente
Al pulsar "Convertir a cliente" en LeadDetail:
1. `INSERT accounts` (name=company, website, sector, status='Activo', health_score=100)
2. `INSERT contacts` (account_id del nuevo account, name, email, origin)
3. `UPDATE leads` SET status='Cualificado', converted_to_contact_id
4. `INSERT activity_log` con action descriptiva
5. Toast + navegar a `/clients/:newId`

### Pipeline Drag & Drop
Al soltar card en nueva columna:
1. `UPDATE deals SET stage = newStage WHERE id = dealId`
2. `INSERT activity_log` con "Deal 'X' movido a [stage]"
3. Toast de confirmación

### Aplicar plantilla SOP → Crear proyecto
Al confirmar en el modal:
1. `INSERT projects` (account_id, name, status='Pendiente', milestones=template.modules, sop_template_id)
2. `INSERT tasks[]` (project_id, title de cada tarea del template, status='Pendiente', hours_budget=4, position=index)
3. `INSERT activity_log`
4. Toast "Proyecto creado con X tareas"

### Cronómetro → Timelog
Al pulsar "Detener":
1. Calcular hours = seconds / 3600 con 2 decimales
2. `INSERT timelogs` (project_id, user_id, description, hours, logged_at=today)
3. Reset timer a 0
4. Toast "Xh registradas"

### Resolver ticket
1. `UPDATE tickets SET status='Resuelto', resolved_at=now() WHERE id=ticketId`
2. `INSERT activity_log` con "[ref] marcado como Resuelto — email enviado"
3. Toast + navegar a `/tickets`

### Aceptar propuesta
1. `UPDATE proposals SET status='Aceptada', accepted_at=now() WHERE id=proposalId`
2. `INSERT activity_log` con "Propuesta '[title]' aceptada por [client]"
3. Toast "Propuesta aceptada · n8n creará el proyecto automáticamente"

### Toggle portal de cliente
1. `UPDATE accounts SET portal_active = !current WHERE id=accountId`
2. Toast

### Retainer — usar vista SQL
En `/maintenance` y en la Vista 360, leer de `v_retainer_status` (NO del campo `retainers.hours_used`):
```typescript
const { data } = await supabase.from('v_retainer_status').select('*').eq('active', true)
```

### Timeline 360 — con fallback
Intentar leer `v_client_timeline` filtrando por `account_id`. Si devuelve 0 resultados, hacer fallback mezclando tickets resueltos + facturas pagadas del mismo account_id ordenados por fecha descendente.

### Realtime — suscripciones activas
Suscribir en estos módulos:
- `leads` (UPDATE audit_status) → actualizar fila en la lista
- `tickets` (INSERT, UPDATE) → actualizar lista y badges
- `n8n_logs` (INSERT) → prepend al feed de logs
- `activity_log` (INSERT) → prepend a la bitácora

### Queries a Supabase con safeQuery wrapper
```typescript
async function safeQuery<T>(query: Promise<{data: T|null, error: any}>): Promise<T|null> {
  const { data, error } = await query;
  if (error) {
    console.error('Supabase error:', error);
    appStore.getState().showToast('Error al cargar datos', '#E24B4A');
    return null;
  }
  return data;
}
```

## 10. PRIORIDAD DE IMPLEMENTACIÓN

Construir en este orden para que cada paso compile sin errores:

1. **Setup:** Vite + Tailwind + dependencias + CSS variables + tipos
2. **Lib:** supabase.ts + utils.ts + constants.ts
3. **Store:** authStore + appStore
4. **Hooks:** useAuth + useToast + useSupabaseQuery + useRealtime + useTimer
5. **Shared components:** MetricCard, Badge, Avatar, ProgressBar, DataTable, Toast, Card, Modal, SkeletonLoader
6. **Layout:** Sidebar + Topbar + Layout + ProtectedRoute + Login
7. **CRM:** Dashboard → Leads/LeadDetail → Pipeline → Contacts → Clients/ClientDetail → Tickets/TicketDetail
8. **Operaciones:** Projects (con SOP) → Maintenance → Tasks → Gantt → Deliverables
9. **Rentabilidad:** TimeTracking → Expenses → Profitability
10. **Informes + Comunicación:** Analytics → Activity → Docs → PortalConfig → PortalPreview
11. **Config:** SopTemplates → Users
12. **n8n:** Automations → N8nLogs → Webhooks
13. **Portal:** ClientPortal (ruta /portal con layout simplificado)

Cada módulo debe:
- Conectar a Supabase con queries reales
- Mostrar SkeletonLoader mientras carga
- Mostrar Toast en acciones de escritura
- Manejar errores con safeQuery
- Ser responsive (sidebar colapsable <768px, tablas con scroll horizontal)

El esquema SQL de Supabase (19 tablas, índices, triggers, RLS, vistas) ya está preparado en el archivo `supabase/schema.sql`. Debe ejecutarse manualmente en el SQL Editor de Supabase ANTES de probar la app.

Los flujos de n8n (5 archivos JSON) están en la carpeta `n8n/`. Se importan manualmente en la instancia de n8n.

La app no necesita backend propio — todo se comunica directamente con Supabase desde el frontend (excepto las automatizaciones que van por n8n).
